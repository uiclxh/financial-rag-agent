# Release 1.0.0

This release records the completed Week 1 retrieval baseline for the Financial RAG Agent project.

## GitHub Release Summary

`v1.0.0` is the first reproducible baseline for an evaluation-first Financial RAG Agent. It focuses on real-data ingestion, strict cleaning, metadata alignment, BM25 retrieval, and retrieval reporting for FinanceBench / SEC filing evidence.

This version is intended to answer one question:

```text
Can the system reliably retrieve the right financial evidence before any answer-generation agent is introduced?
```

The answer for the current benchmark is yes at `top_k=10`: the validated run reaches full evidence recall, full question recall, full gold-page hit, and zero wrong-company / wrong-year retrieval errors.

## Scope

Week 1 covers ingestion, strict cleaning, BM25 retrieval, retrieval evaluation, and reporting. It intentionally does not include answer generation, citation verification, vector retrieval, reranking, or valuation.

## How to Read These Metrics

- `evidence_level_retrieval_recall@10 = 1.000` means all required gold evidence units were retrieved in the top 10 for answerable questions.
- `question_recall@10 = 1.000` means every answerable question had all required evidence in the top 10.
- `gold_page_hit@10 = 1.000` means the retrieved results included the gold evidence page for every answerable question.
- `strict_precision@10` is low by design because the corpus contains many distractor evidence pages and precision counts only required gold evidence.
- `relaxed_precision@10` also counts equivalent or supporting evidence.
- `wrong_company_rate` and `wrong_year_rate` track high-risk financial retrieval errors.

## Reproduce

```bash
python -m src.data_acquisition.download_real --output-dir data/raw/real --sec-tickers MMM,AMD,BBY,AMCR,PEP,BA,PFE,ADBE,VZ,GLW,CVS,GIS,MGM,NKE,AES,AMZN,AWK,XYZ,KO,JNJ,LMT,WMT,MSFT,NFLX,ULTA,COST,KHC
python -m src.ingestion.financebench --config configs/ingestion/financebench_real.yaml
python -m src.retrieval.index --config configs/retrieval/bm25_financebench_real.yaml
python -m src.evaluation.run --config configs/evaluation/financebench_real_week1.yaml
python -m src.reports.build --run-id latest
```

Or run the validation helper:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\validate_week1.ps1
```

## Not Included Yet

The following are intentionally out of scope for `v1.0.0`:

- answer generation
- material claim extraction
- citation verification
- hallucination and faithfulness scoring
- vector retrieval
- reranking
- XBRL calculation workflows
- valuation workflows
- production service deployment

## Known Limitations

- The current real-data corpus uses FinanceBench evidence pages as retrieval chunks rather than full SEC filing parsing.
- Unanswerable questions are included in the benchmark, but refusal quality is not evaluated until the answer-generation phase.
- Precision is expected to improve after Week 2 vector retrieval and reranking.
- This release validates retrieval behavior only; it does not make claims about final answer quality.

## Next Milestone

Week 2 will add vector retrieval and reranking, then compare retrieval recall / precision against the BM25-only baseline.
